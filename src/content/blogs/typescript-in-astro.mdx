# TypeScript in Astro

Astro ships with TypeScript support out of the box — no config needed.

## Typed Props

```astro
---
interface Props {
  title: string;
  count?: number;
  tags: string[];
}
const { title, count = 0, tags } = Astro.props;
---
```

## Typed Content Collections

```ts
// src/content/config.ts
import { defineCollection, z } from 'astro:content';

const blog = defineCollection({
  schema: z.object({
    title: z.string(),
    tags: z.array(z.string()),
    pubDate: z.coerce.date(),
  }),
});
```

## Why It Matters

Type safety catches bugs at build time, not in production. With Zod schemas, your frontmatter is validated automatically.
